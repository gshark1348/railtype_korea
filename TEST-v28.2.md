- 무한모드 운행 구간별 좌표 경로 저장

- 출발·환승·운행 종료 지점 표시
- 노선 설정 화면 모션 데모 재생 복구
- 데스크톱 한 화면 게임 레이아웃

## 무한 여정 지도 구조

- 각 운행 구간은 `path` 배열에 역명·역 코드·위도·경도를 저장합니다.
- 정답 입력 후 열차가 다음 입력 대상역에 도착하면 해당 좌표를 현재 구간에 추가합니다.
- 환승과 자동 회차는 새 구간을 만들고, 환승역부터 새 노선색으로 경로를 기록합니다.
- 운행 종료 시 경로 배열을 결과 객체에 깊은 복사해 현재 게임 상태와 분리합니다.
- 결과 SVG는 모든 구간을 같은 지리 투영에 배치하고 노선색별 polyline으로 표시합니다.
- 시작점, 환승점, 마지막 운행 종료역에 별도 마커와 라벨을 표시합니다.

## 모션 데모 복구

기존에는 `display:none` 상태의 설정 모달 안에서 SVG path 길이를 계산해 일부 브라우저에서 길이가 0이 될 수 있었습니다.

v28.2에서는 다음 순서로 동작합니다.

1. 설정 모달 표시
2. 두 번의 `requestAnimationFrame`으로 실제 레이아웃 배치 대기
3. 데모 SVG 지도 재생성
4. 경로 길이 유효성 확인
5. 모션 데모 자동 재생

재생 버튼을 눌렀을 때 경로 길이가 유효하지 않으면 지도를 다시 생성하는 보조 복구 로직도 포함합니다.

## 한 화면 게임 UI

1081px 이상의 데스크톱 화면에서는 다음 요소가 한 뷰포트 안에 배치됩니다.

- 지리 노선 지도
- 현재 입력 대상역과 지역 설명
- 타자 입력창과 피드백
- 실시간 시간·정확도·오답
- 무한모드 환승 현황
- 현재 위치 주변 역
- 운행 종료 버튼

1080px 이하에서는 기존 세로형 반응형 레이아웃을 사용합니다.

## 참고

현재 실행 환경의 Chromium에는 로컬 URL과 data URL을 차단하는 조직 정책이 적용되어 자동 브라우저 캡처를 수행할 수 없었습니다. 대신 JavaScript 구문, HTML 참조, 중복 ID, CSS 구조와 상태 전환 코드를 정적으로 검증했습니다.
